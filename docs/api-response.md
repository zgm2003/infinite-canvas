# 前后端接口响应约定

后端业务接口统一返回 JSON：

```json
{
  "code": 0,
  "data": {},
  "msg": "ok"
}
```

- `code`: 业务状态码，`0` 表示成功，非 `0` 表示失败。
- `data`: 业务数据。失败时通常为 `null`。
- `msg`: 响应消息。成功默认为 `ok`，失败时放错误原因。

前端请求逻辑必须同时看 HTTP 状态码和 `code`。

- 普通业务失败仍可能返回 HTTP 200 + `code != 0`，这是兼容旧接口的保守行为。
- 已明确语义的网关层失败会返回对应 HTTP 状态，同时保留 `{ code, data, msg }` 结构，例如未登录 `401`、权限不足 `403`、请求参数错误 `400`、上游 AI 接口失败 `502`。
- service 层错误开始支持 typed status；已经携带状态的错误会通过 `FailError` 返回对应 HTTP 状态，typed error 被 `%w` 包装时也会保留安全文案和状态码。例如登录用户名或密码错误返回 `401`，账号禁用返回 `403`，注册或后台保存用户的参数错误返回 `400`，用户名冲突返回 `409`，编辑不存在用户返回 `404`，Linux.do OAuth 未开启返回 `403`、未配置返回 `400`、上游登录失败返回 `502`，AI 生成算力点不足返回 `402`，AI 退款用户缺失返回 `404`，后台渠道缺接口地址、API Key 或测试模型名返回 `400`，后台渠道网络错误、上游 HTTP 错误、HTTP 200 JSON 业务错误或兜底上游错误返回 `502`。尚未 typed 的旧业务错误仍可能保持 HTTP `200` + `code != 0`。
- AI 代理会限制客户端传入的生成数量 `n`，并在算力点乘法前做溢出防护；负数算力点消费属于参数错误。
- OpenAI 兼容代理的成功响应通常透传上游；但云端渠道 POST 已开始识别上游 HTTP 200 JSON 里的 `error`、顶层数字或字符串 `code` 非成功值、`status=error/failed/cancelled`、`success=false` 或视频创建缺任务 ID，并把这类业务失败映射为 HTTP `502` 以便触发退款。上游漏写 JSON `Content-Type` 或误写成 `text/plain` 但响应体是 JSON 错误时，也会按业务失败处理；`stream=true` 请求不会为了探测成功响应预读 SSE 长连接，且请求体超过 64KB 时仍能识别 `stream=true`；如果流式请求返回有明确长度的短 JSON 错误体，仍会按业务失败处理并触发退款；视频任务创建/轮询这类必须校验 JSON 体的路径，同样不依赖上游正确设置 JSON `Content-Type`。
- 前端不要只依赖 HTTP 状态码，也不要只依赖 `code`；统一取 `msg` 作为可读错误信息。

接口连接失败、服务不可达、返回体不是约定 JSON 时，前端按网络或接口异常处理。
